# HabitContract 功能说明文档

## 1. 业务背景

- 自律困难者难以坚持习惯，传统打卡缺乏约束力
- 契约模式引入社交压力和轻惩罚，提高坚持概率
- 通过将习惯养成转化为契约关系，借助监督伙伴的外部压力和违约惩罚的内部驱动，双重保障习惯的持续执行

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| 契约所有者（Owner） | 创建契约、管理违约记录、邀请监督伙伴 |
| 监督伙伴（Partner） | 接受邀请、监督契约执行、可接受或拒绝邀请 |

## 3. 核心用例

- 用户注册/登录
- 创建习惯契约（设定目标、频率、惩罚）
- 邀请监督伙伴
- 每日打卡
- 违约记录
- 查看统计数据

## 4. 功能模块详细说明

### 4.1 用户认证模块

| 功能 | 说明 |
|------|------|
| 注册 | 用户名 + 邮箱 + 密码，密码使用 BCrypt 加密存储 |
| 登录 | 支持用户名或邮箱登录，返回 JWT Token |
| 个人信息 | 查看和更新当前用户信息 |

### 4.2 契约管理模块

| 功能 | 说明 |
|------|------|
| CRUD | 创建、查询、更新、删除契约，支持分页、搜索、排序 |
| 状态流转 | Active ↔ Paused，Active → Completed / Failed |
| 权限控制 | 仅契约所有者可操作 |

**契约状态说明：**

| 状态 | 说明 |
|------|------|
| Active | 进行中，可正常打卡 |
| Paused | 已暂停，暂停期间不可打卡 |
| Completed | 已完成，契约目标达成 |
| Failed | 已失败，契约未能完成 |

### 4.3 监督伙伴模块

| 功能 | 说明 |
|------|------|
| 邀请伙伴 | 契约所有者邀请其他用户作为监督伙伴，状态为 Pending |
| 接受/拒绝 | 伙伴可接受或拒绝邀请 |
| 权限控制 | 仅契约所有者可邀请 |

**伙伴状态说明：**

| 状态 | 说明 |
|------|------|
| Pending | 待接受，已发出邀请 |
| Accepted | 已接受，正在监督 |
| Rejected | 已拒绝 |

### 4.4 打卡模块

| 功能 | 说明 |
|------|------|
| 创建打卡 | 需契约为 Active 状态，支持文字证明 |
| 修改打卡 | 仅创建者可修改 |
| 我的打卡 | 按当前用户筛选打卡记录 |

### 4.5 违约模块

| 功能 | 说明 |
|------|------|
| 记录违约 | 记录违约原因和日期 |
| 确认违约 | 标记违约是否已确认 |
| 权限控制 | 仅契约所有者可管理 |

### 4.6 统计模块

| 功能 | 说明 |
|------|------|
| 总览统计 | 用户数、契约数、打卡数、违约数、完成率 |
| 趋势统计 | 按日统计，支持日期范围筛选 |

## 5. 数据库 ER 图文字描述

```
Users(1) ──→ (N) Contracts        一个用户可创建多个契约
Users(1) ──→ (N) CheckIns         一个用户有多条打卡记录
Users(1) ──→ (N) ContractPartners 一个用户可作为多个契约的伙伴
Contracts(1) ──→ (N) CheckIns            一个契约有多条打卡记录
Contracts(1) ──→ (N) ContractPartners    一个契约可有多个监督伙伴
Contracts(1) ──→ (N) ContractViolations  一个契约可有多条违约记录
```

## 6. 关键业务规则

- **契约状态流转**：Active → Paused（暂停），Paused → Active（恢复），Active → Completed（完成），Active → Failed（失败）
- **伙伴状态流转**：Pending → Accepted（接受），Pending → Rejected（拒绝）
- **打卡时契约必须为 Active 状态**
- **仅契约所有者可邀请伙伴和管理违约**
- **伙伴可接受或拒绝邀请**

## 7. 接口调用示例

### 7.1 用户注册

```
POST /api/auth/register
```

```json
{
  "username": "testuser",
  "email": "test@example.com",
  "password": "Test@123"
}
```

### 7.2 创建契约

```
POST /api/contracts
Authorization: Bearer <token>
```

```json
{
  "habitName": "每天早起6点",
  "frequency": "每天",
  "startDate": "2026-01-01",
  "endDate": "2026-12-31",
  "penaltyDescription": "发红包50元给监督伙伴"
}
```

### 7.3 打卡

```
POST /api/checkins
Authorization: Bearer <token>
```

```json
{
  "contractId": 1,
  "checkInDate": "2026-06-20",
  "proofText": "今天6点准时起床！"
}
```
